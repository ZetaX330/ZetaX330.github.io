本文记录本博客基于 Hexo + Butterfly 主题的完整操作流程，方便日后查阅。

项目概况

项目 详情
框架 Hexo v8.x
主题 Butterfly v5.x（通过 npm 安装）
部署 GitHub Pages → https://zetax330.github.io
语言 简体中文 (zh-CN)

目录结构

常用命令

命令 作用
hexo new "文章标题" 创建新文章，生成到 source/_posts/
hexo new page "页面名" 创建新页面（如「关于」），生成到 source/页面名/
hexo new draft "草稿名" 创建草稿 → source/_drafts/
hexo publish "草稿名" 将草稿正式发布
npm run server 启动本地预览 → http://localhost:4000
hexo server --draft 本地预览时也显示草稿
npm run build 生成静态文件到 public/
npm run deploy 构建并部署到 GitHub Pages
npm run clean 清除缓存和构建产物

主题自定义

Butterfly 主题通过 npm 安装，位于 node_modules/hexo-theme-butterfly/。

要覆盖默认设置，在项目根目录创建 _config.butterfly.yml 文件，写入自定义项。常用配置：

  • 导航菜单 (menu) — 首页、归档、分类、标签、关于
  • 头像 (avatar) — 如 /img/avatar.png
  • 社交链接 (social) — GitHub、邮箱等图标
  • 代码高亮主题 (highlight_theme)
  • 评论系统 (comments) — Gitalk / Disqus / Waline 等

主题详细配置参考 Butterfly 官方文档。

配置文件速查

文件 作用
_config.yml Hexo 网站级配置（标题、URL、部署等）
_config.butterfly.yml Butterfly 主题自定义（需自行创建）
_config.landscape.yml 旧主题遗留文件（空文件，可删除）
package.json npm 依赖声明和脚本命令
scaffolds/*.md 新文章/页面的内容模板

常见问题

本地预览内容不对

先运行 npm run clean 清除缓存，再 npm run server 重新启动。

部署后网站没更新

检查 _config.yml 中 deploy 段的仓库地址和分支：

确认本地 git 能正常推送到该远程仓库。

想把文章的图片放哪里

两种方式：

  1. 放在 source/img/ — 文章中引用 ![](/img/xxx.png)
  2. 开启文章资源文件夹 — 在 _config.yml 中设置 post_asset_folder: true，然后 hexo new 时会同时生成同名文件夹，图片放里面即可
